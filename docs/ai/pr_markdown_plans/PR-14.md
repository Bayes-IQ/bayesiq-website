## Goal

Add lightweight `/privacy` and `/terms` pages to establish trust with technical buyers and improve email deliverability posture. These are minimal but real pages — not placeholder stubs. They should include company identification and contact information. Both pages should be linked from the footer.

---

## Scope

### Files Added

| File | Purpose |
|------|---------|
| `src/app/privacy/page.tsx` | Privacy policy page with data handling, retention, contact info |
| `src/app/terms/page.tsx` | Terms of service page with engagement terms, liability, IP |

### Files Modified

| File | What changes |
|------|-------------|
| `src/components/Footer.tsx` | Add Privacy and Terms links |
| `site.config.yaml` | Add page entries with metadata |

### Non-goals

- No cookie consent banner (not needed — Vercel Analytics is cookie-free)
- No `/security` page (deferred, potentially Phase 3)
- No legal review — these are starting-point pages to be reviewed by counsel later

---

## Plan

1. **[P1] Create `/privacy` route**
   - New `src/app/privacy/page.tsx`
   - Content: what data we collect (contact form submissions only), how we use it, retention, third parties (Vercel, Resend), contact email for data requests
   - Export metadata with title "Privacy Policy" and appropriate description

2. **[P2] Create `/terms` route**
   - New `src/app/terms/page.tsx`
   - Content: scope of services, engagement terms, intellectual property, limitation of liability, governing law placeholder
   - Export metadata with title "Terms of Service" and appropriate description

3. **[P3] Update Footer with links**
   - Add Privacy Policy and Terms of Service links to Footer component

4. **[P4] Update site.config.yaml**
   - Add entries for /privacy and /terms with titles and descriptions
